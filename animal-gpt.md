# 動物守護者（暫名）— MVP 設計計畫（animal-gpt）

本文件是把 `animal-fight-prompt.txt` 的需求「落地成可產出的規格」：定義 MVP 規則的固定選項、內容生成順序、數值平衡準則與 A4 Print & Play 版面規範，讓後續可以一致地產出 Part 1–4。

## 1. 目標與受眾
- 玩家：2 位孩子（約 6 歲偏圖像直覺、10 歲偏策略計算）
- 引導者：1 位家長擔任 DM（負責敘事、裁決優勢/劣勢、帶節奏）
- 核心體驗：合作討論 → 選擇誰出手/用哪個屬性 → 擲骰檢定 → 故事推進

## 2. MVP 範圍（做什麼／不做什麼）
- 做：屬性檢定、共享生命值（愛心）、前進標記、雙 6/雙 1、Boss 戰與逃脫模式、10 張動物卡、3 組 d6 表、A4 排版指引
- 不做：裝備、複雜魔法/法術表、多資源管理、長篇戰鬥數值系統

## 3. 規則定稿（MVP 需固定，避免輸出歧義）

### 3.1 基本檢定
- 擲骰：`2d6 + 對應屬性數值`
- 標準難度：`DC 12`
- 成功：`結果 >= DC` → 取得 `1` 枚前進標記（並由 DM 描述好結果）
- 失敗：`結果 < DC` → 扣 `1` 顆愛心（並由 DM 描述代價/挫折）
- 大成功：骰出「雙 6」→ 直接成功，取得 `2` 枚前進標記
- 大失敗：骰出「雙 1」→ 直接失敗，扣 `2` 顆愛心

### 3.2 優勢／劣勢（採「調整 DC」以降低孩子算術負擔）
- 優勢：`DC 10`
- 劣勢：`DC 14`
- DM 裁決原則（寫進 Part 1）：
  - 鼓勵孩子說明「為什麼用這個動物/屬性很合理」→ 給優勢
  - 明顯不利（下雨濕滑爬樹、在狹縫用大體型硬擠等）→ 給劣勢

### 3.3 共享生命值與節奏
- 隊伍共享 `5` 顆愛心（建議用實體標記）
- 目標：累積 `5` 枚前進標記後進入終局

### 3.4 終局（同時提供兩種模式，讓 DM 依孩子狀態選）
- Boss 戰（較像「硬碰硬」）：
  - 共 `3` 輪，每輪 1 次檢定
  - 建議難度：`DC 14`
  - 3 輪中成功次數 `>= 2` → 勝利；否則再扣 `1` 顆愛心後由 DM 決定是否「撤退進逃脫模式」
- 逃脫模式（較像「追逐解障礙」）：
  - 連續 `3` 關：每關擲 `1d6` 查「逃脫障礙表」→ 指定對應屬性
  - 每關成功就通過；任一關失敗扣愛心並繼續下一關
  - 3 關結束愛心未歸零 → 勝利

## 4. 交付內容（對應 prompt 的 Part 1–4）

### Part 1：給 DM 的規則摘要卡（One-Pager）
輸出目標：家長「邊玩邊看」不迷路。
- 必備區塊：
  - 遊戲準備（選卡、放 5 愛心、準備 2d6）
  - 回合流程（DM 描述 → 選人選屬性 → 擲骰加值 → 結算）
  - 成功/失敗/雙 6/雙 1 的效果（表格化）
  - 優勢/劣勢（DC 10/12/14 + 裁決提示語）
  - 終局（Boss/逃脫各 3 行以內的流程）
  - 屬性對照（1–6 對應圖示與用途示例）
- 文案語氣：用「你可以問孩子…」的引導句，鼓勵共創故事。

### Part 2：動物卡牌資料庫（10 張，Markdown 表格）
輸出目標：6 歲靠圖示找格子、10 歲看數值做決策；沒有絕對強卡。
- 欄位固定：
  - 動物名稱（具體物種）
  - Slot 1–5：力量/速度/攻擊/防禦/智慧（各 `1–9`）
  - Slot 6：天賦技能（名稱 + 數值 `10–12`）
  - 總和檢查：`(1–5 合計) + (天賦數值) = 32–38`
  - 視覺建議：給 6 歲的「3 個可被畫出來的特徵」
- 平衡準則（做卡時逐條檢核）：
  - 每張至少 1 個弱項（`<= 3`）
  - 每張至少 1 個亮點（Slot 1–5 至少 1 個 `>= 8` 或天賦 `>= 11`）
  - 10 張覆蓋不同原型（建議各 2 張）：
    - 坦克（高防/高力、低速）
    - 速度（高速、低防）
    - 輸出（高攻、偏脆）
    - 智慧（高智、靠情境取勝）
    - 全能（中庸但天賦強）
- 天賦設計規則（MVP 先「敘事觸發」即可）：
  - 名稱要能被孩子喊得出來（2–4 字動作感）
  - 對應動物真實特徵（象=震地、貓頭鷹=夜視、電鰻=放電…）

### Part 3：劇本與情境表（Scenario Tables）
輸出目標：DM 只要擲骰就有故事線與清楚的機制。
- 開局情境表（d6）：每列包含「動機 + 任務目標 + 一句開場旁白」
- 突發狀況表（d6）：每列包含「環境描述 + 明確機制」
  - 機制建議：`DC +2 / DC -2`、或「下一次檢定給優勢/劣勢」、或「回復/扣 1 愛心」
- 逃脫模式障礙表（d6）：
  - 點數 1–6 必須一一對應 Slot 1–6
  - 每列給 1 句障礙描述 + 2 個可行解法提示（讓孩子選擇）

### Part 4：A4 版面設計指引（Layout Guide）
輸出目標：A4 雷射列印、裁切容易、6 歲能「看圖就知道看哪格」。
- A4 排版選項（二選一，MVP 任選一種並固定）：
  - 8 張/頁：`4 x 2`，卡片約 `63 x 88mm`，邊距 `8–10mm`
  - 9 張/頁：`3 x 3`，卡片略小但更省紙
- 卡牌框架（UI）固定結構：
  - 上：動物名（大字）+ 插圖區（約 40% 高度）
  - 中：`2 行 x 3 列` 的屬性格（每格都要有「骰點圖 + 屬性 icon + 大數字」）
  - 下：天賦（星星 icon + 名稱 + 大數字）+ 一句短台詞（可選）
- 視覺引導（給 6 歲）：
  - 骰點圖（1–6）要比文字更醒目；孩子擲出 3 就去找「三點骰」那格
  - 數字用大字粗體（建議 `24–32pt`）
  - 顏色編碼（可選）：力紅／速綠／攻橘／防藍／智紫／天賦金
- 印製建議：160–200gsm；或普通紙 + 卡套 + 廢牌襯底。

## 5. 產出流程（建議順序，避免返工）
1. 先把第 3 節「規則定稿」寫成 Part 1 One-Pager（確定 DC、雙 6/雙 1、終局節奏）
2. 依「原型配比」先列出 10 個物種清單 → 再填數值 → 最後補天賦與視覺特徵
3. 以「需要哪些屬性」反推情境表：確保 1–6 都常見、且有讓弱項也能發揮的局
4. 依 UI 規格做 A4 版面：先做 1 張卡樣板，再批量套版

## 6. 驗收與平衡檢核（MVP Done Definition）
- 可教學：DM 讀 One-Pager 2 分鐘內能開局
- 可完成：平均 15–20 分鐘能結束一局
- 成功率：在標準 DC 12 下，多數挑戰成功率約 40–70%（孩子會覺得緊張但可贏）
- 互補性：任兩張卡都能找到「誰適合處理這關」的討論空間
- 可讀性：6 歲能在 3 秒內靠骰點圖找到對應屬性格

## 7. 後續擴充（不納入 MVP）
- 裝備卡（一次性 +1/+2）、狀態（暈眩、燃燒）、更多動物與地區套件、Boss 專屬招式表
