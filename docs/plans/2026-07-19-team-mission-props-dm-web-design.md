## 為什麼調整

原本 A4 控制板同時放護盾、後援、關卡進度、三個動物區與規則速查，資訊過密，也讓孩子較難把桌面狀態和規則說明分開。後續又將護盾改成三件可抽取、只在 Boss 使用的寶物，讓一般任務成功後有更具體的收穫。

本版拆成三個用途：

1. 孩子會操作的狀態與區域，改成可翻面或放在桌上的 Poker 卡。
2. 孩子會看的劇情需求，放在大型揭露卡，以插圖、屬性小圖與數字為主。
3. DM 私下使用的規則、旁白與測試紀錄，移到手機／平板網頁。

## 一、8 張狀態與區域卡

使用 2 張 A4 四分雷射標籤紙，每格 105×148.5mm，直接貼到 122×175mm Poker，不裁切。

| 類型 | 張數 | 桌面操作 |
|---|---:|---|
| 萬用背包 | 1 | 與另外兩件寶物洗勻、背面朝上；一般任務成功時抽出，Boss 時增加並分配骰子 |
| 探險手電筒 | 1 | 同上；Boss 時照出弱點並降低門檻 |
| 變化魔鏡 | 1 | 同上；Boss 支援後調整骰面，三顆同號觸發魔鏡反轉 |
| 全隊後援 | 2 | 開局正面朝上；呼叫後援時翻面 1 張 |
| 待命區 | 1 | Poker 橫放；尚未出動的動物卡放在旁邊 |
| 再挑戰區 | 1 | Poker 橫放；第一次出動為 0 成功的動物卡放在旁邊 |
| 休息區 | 1 | Poker 橫放；已完成工作或用完機會的動物卡放在旁邊 |

動物卡不疊在區域 Poker 上。區域 Poker 只作為桌面分區標題。

### 視覺規格

- 全套黑白雷射列印，不以顏色區分。
- 三張寶物卡使用粗實線外框；後援卡使用雙線外框。
- 待命區使用 `0.8mm` 實線、再挑戰區使用 `1.2mm` 粗虛線、休息區使用 `1.7mm` 粗點線，確保黑白列印後仍能明顯分辨。
- 三張區域卡旋轉後的外框統一保留四周 `5mm` 安全邊界，避免標籤紙與印表機誤差切到框線。
- 中文標題使用全卡注音圓體 Bold；一句說明使用圓體 Medium。
- 圖像只輔助辨識，不承載獨立規則。
- 三區固定以水豚、狐獴、巨嘴鳥組成示意隊伍，三張都出現完整隊伍並以明顯不同的動作區分狀態；必要時加入入口、側邊小徑或安全遮蔭處等最低限度背景，但不綁定特定劇情。採 V2 圖鑑平衡版黑白插畫。
- 寶物圖採「孩子熟悉的物件＋一點神奇能力」，不綁定特定 Boss：萬用背包為直立方形、探險手電筒為斜向物件加三角光束、變化魔鏡為直立橢圓。單一主物件占圖區約 65–75%，白底、五階灰階、中粗輪廓，不畫完整背景。
- 萬用背包打開並露出骰子與少量通用工具；探險手電筒照出障礙上的入口或薄弱位置；變化魔鏡呈現鏡外失敗骰、鏡內骰面 4，鏡冠放三顆同號骰。
- 寶物共同孩子口訣：「背包拿工具，手電筒找弱點，魔鏡變骰子。」完整規則仍以數字、骰面小圖與箭頭呈現，插圖不獨自承載規則。

## 二、4 張大型劇情揭露卡

使用 2 張 A4 二分雷射標籤紙。每一格 210×148.5mm，內含一張 170×118.5mm 卡面，裁切後貼到 122×175mm Poker。

揭露卡採橫式版面，讓情境圖、屬性需求與數字可以在同一視線內完成辨識。

固定四張：

1. 任務簡報＋Boss 預告。
2. 第一關單動物任務。
3. 第二關雙動物任務。
4. 第三關單動物任務。

Boss 抵達時重新查看第 1 張，不增加第五張 Boss 卡。

### 資訊優先順序

1. 大型黑白情境插圖：只呈現環境問題，不暗示應選哪隻玩家動物。
2. 工作名稱與一句白話原因。
3. 置中的需求數字牌，直接列出所有成功骰面，例如 `7、8、9`。
4. 兩張與動物卡相同的屬性小插圖。
5. 屬性名稱與注音，供識字孩子閱讀。

每個工作區塊採單一路徑由上往下閱讀：情境圖片 → 一句白話原因 → 需求數字 → 兩個可選屬性。數字不再寫成 `7-9` 或 `7 – 9`，改為 `7、8、9`，讓 5 歲孩子直接看出三個可接受的骰面。數字置於淺灰圓角牌中，不使用橫向分隔線。

兩個屬性左右並排，以小型黑底「或」標記表示選擇關係；不加個別屬性外框。屬性圖片在上、名稱與注音在圖片正下方，與動物卡片的閱讀方向一致。單動物屬性組置中並限制為 `96mm` 寬，避免兩張圖片距離過遠；雙動物任務各自在 2A／2B 分區內使用同一結構。

此順序同時適用單動物、雙動物與開場 Boss 預告，不得改回「屬性在左、數字在右」。單動物 `96mm` 內縮與雙動物版本均已確認，並已遷移到正式 4 張揭露卡。

不再顯示「小格／剛好格／大格」。卡片直接寫：

> 力量或攻擊 `7、8、9`

並補上因果句：

> 因為要突破厚重水牆。

### 第二關雙工作

共用一張藤橋情境圖，下方分成 2A、2B 兩個明確區塊。每個區塊各自顯示工作名稱、一句白話原因、圓角需求數字牌，以及下排兩個「圖片在上、名稱在下」的屬性選項；孩子先看完整場景，再分配兩隻動物。

## 三、DM 專注模式網頁

部署目標為 Cloudflare Pages，可由手機、平板或電腦開啟。MVP 使用純靜態 HTML／CSS／JavaScript，不需要帳號或後端。

### 操作流程

1. 首頁選擇劇情與遊玩模式。
2. 進入開場、第一關、第二關、第三關、Boss、結局等步驟。
3. 畫面一次只顯示當下步驟；上一步／下一步固定在容易按的位置。
4. 大字「直接念」區與較小的「DM 私下操作」區分開。
5. 骰池、三件寶物、Boss 門檻、支援與後援規則只在需要時出現。
6. 目前進度自動存在 `localStorage`，重新整理後可繼續。

寶物與後援以桌面實體卡為當局唯一狀態來源，網頁不要求遊戲中逐骰記帳；結束摘要才記錄測試資料。

### Boss 專注流程

Boss 畫面固定依序顯示：

1. 探險手電筒：4d6 數奇數，確定門檻。
2. 變化魔鏡：3d6 記下最大減最小，或三顆同號的魔鏡反轉。
3. 萬用背包：2d6 相加，決定額外骰並分給 A、B。
4. 擲 A、B 主要骰。
5. 單向支援一次。
6. 分配魔鏡調整點或使用反轉。
7. 檢查 Boss 門檻。
8. 仍失敗才檢查全隊後援；後援與寶物後結果取較高、不相加。

### MVP 測試紀錄

遊戲中不要求逐骰輸入。結束時填寫約 30 秒的摘要：

- 劇情 ID 與版本（自動）
- 遊玩模式
- 開始、結束與總時間（自動）
- 第一、二、三關各自成功／失敗
- 取得寶物 0–3 件及寶物名稱
- Boss 原始骰池 A／B（各 1–3 顆）
- 背包增加骰數、手電筒降低級數、魔鏡點數或反轉
- 已用後援 0–2
- Boss 是否在後援前已成功
- Boss 是否完成
- 孩子理解程度
- DM 操作負擔
- 一行備註

紀錄只存在本機，可複製 Markdown 或匯出 JSON。MVP 不使用 Pages Functions、D1、登入或跨裝置同步。
