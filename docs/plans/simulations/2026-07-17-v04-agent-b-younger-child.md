### 10. 遊戲結束後分類

**阻擋問題**

- 無。所有指定邊界都能依 v0.4 得出唯一結果，沒有循環、重複計算或成功數相加歧義。

**非阻擋問題**

1. 單動物前線失敗後，後援仍可能 0 成功並立即消耗；本局第一關連續出現兩次完整失敗，雖然規則清楚，但親子實玩時可能形成較強的雙重挫折。
2. 3 面盾時，1 顆骰的 Boss 前線角色在擲骰前便確定無法獨自達成 2 成功。規則可以清楚說明，但因 v0.4 不允許前線／後援換位，孩子仍必須執行一個已知不能單獨達標的動作。
3. 「只提升一邊是否可能通關」需要 DM 同時查看目前分布與 Boss 門檻。本局 `0＋0` 與三盾 `1＋1` 都能唯一判斷，但比一般單動物補救多一層反事實計算。

**純觀察**

1. 協定同時要求「進 Boss 時仍有 3 面盾」與「結局交代已完成的救援」。依 v0.4，3 面盾代表前三個任務沒有任何一關機械成功，因此本局只能保留失敗旁白中的局部救援成果，不能宣稱有完整任務成功。
2. 第一關使用後援、第三關保留後援，確實產生兩種不同的後援決策；後援不是自動使用。
3. Task 2 的 `0＋0` 與 Boss 的 `1＋1／需 2＋2` 都正確阻止後援，沒有消耗仍留在後援區的食人魚。
4. 失敗旁白沒有使用「你選錯」「這隻太弱」「方法不對」等歸責語句。
