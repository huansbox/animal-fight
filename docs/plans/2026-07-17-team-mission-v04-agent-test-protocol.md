# 團隊任務 v0.4：Agent 模擬實測協定

> 日期：2026-07-17
> 狀態：測試規則凍結，等待 3 agents 完成 6 局
> 規則來源：[`2026-07-15-team-mission-v04-rules-review.md`](2026-07-15-team-mission-v04-rules-review.md) 第一至九節
> 動物資料：[`game/digital/data/animals.json`](../../game/digital/data/animals.json)

## 一、目的與限制

本輪只測 v0.4 能否一致執行，不在模擬途中修改規則。每個 agent 完成一局自然隨機局與一局邊界壓力局，共 6 局。

Agent 模擬可以找出規則歧義、計算錯誤、無效選擇與 DM 操作負擔，但不能證明孩子真的看得懂或喜歡。通過本輪後只能稱為「實玩候選規則」，仍需日後親子實玩。

## 二、固定任務骨架

六個位置開局全部公開：

| 位置 | 故事需求 | 區間 | 可選屬性 |
|---|---|---|---|
| 1. 尋找安全道路 | 在倒木與積水間找出可通行路線 | 剛好格 4–6 | 速度／智慧 |
| 2A. 固定藤橋 | 抵抗水流並拉緊藤橋 | 大格 7–9 | 力量／防禦 |
| 2B. 接近幼獸 | 輕輕通過脆弱橋面，不驚嚇幼獸 | 小格 1–3 | 速度／攻擊 |
| 3. 搬運藥品 | 輕拿藥箱，避免玻璃瓶破裂 | 小格 1–3 | 力量／速度 |
| 4A. 突破水牆 | 打開洪水巨獸的水牆 | 大格 7–9 | 力量／攻擊 |
| 4B. 保護動物 | 看準水流並穩定保護避難區 | 剛好格 4–6 | 防禦／智慧 |

小格、剛好格、大格各 2 個位置。Boss 開始有 3 面盾，前三關每成功一關打破 1 面。

## 三、自然隨機局

### 可重現亂數

使用 Python `random.Random(seed)`：

1. `rng.sample(animals, 8)` 抽出 8 隻動物。
2. 玩家完成六個前線位置與兩隻後援配置後，才開始擲骰。
3. 每次需要骰子時，依遊戲流程呼叫 `rng.randint(1, 6)`；重擲與後援也接續消耗同一 RNG。
4. 故事描述與玩家決策不消耗 RNG。

固定抽牌：

| Agent | Seed | 玩家模型 | 抽出的 8 隻動物 |
|---|---:|---|---|
| A | 4101 | 9 歲、會找最佳配置 | 鴨嘴獸、乳牛、狼蛛、灣鱷、長頸鹿、北極熊、犰狳、裸鼴鼠 |
| B | 5202 | 5 歲、先選喜歡與直覺合理的動物 | 鯨頭鸛、東部灰松鼠、針鼴、大白鯊、加拉巴哥象龜、浣熊、食人魚、狐獴 |
| C | 6303 | 兄弟合作，各控制 4 張 | 蜂鳥、黑猩猩、海獺、大熊貓、浣熊、灰狼、長毛象、蟒蛇 |

### 玩家行為

- Agent A：可以完整計算六格的骰池，選擇自認最佳配置，但不能偷看骰子。
- Agent B：先依圖片、動物喜好與故事直覺選擇；卡住時 DM 只給兩個候選，不直接說最佳答案。
- Agent C：抽牌順序前 4 張交給哥哥、後 4 張交給弟弟。每人配置 3 隻前線與 1 隻後援；另一人只能提供一次建議。

## 四、邊界壓力局

邊界局仍必須使用現有動物資料，但可以刻意選 8 隻動物與安排骰序，以確保測項真的出現。骰序必須在各關擲骰前公開記錄，不得看到結果後補改。

### Agent A：規則與策略邊界

完整一局中至少包含：

1. 一般雙動物任務初始未過關，由 `2＋0` 或 `3＋0` 觸發單向支援。
2. Boss 兩邊都有成功但總數不足，使用後援替換其中一邊並取較高值。
3. 玩家保留至少 1 隻後援到 Boss。
4. 檢查開局最佳配置是否過於明顯，或是否有多個合理配置。

### Agent B：失敗與補救邊界

完整一局中至少包含：

1. 單動物 0 成功，玩家必須決定接受失敗或消耗後援。
2. 雙動物兩邊都無法靠只提升 1 隻而通關，因此不能叫後援。
3. 進 Boss 時仍有 3 面盾。
4. Boss 至少一個位置只有 1 顆骰，驗證「需要 2 成功但物理上不可能」是否能被清楚說明。
5. 故事結局仍交代已完成的救援，不把失敗歸咎於孩子的想法。

### Agent C：雙孩子操作邊界

完整一局中至少包含：

1. 兩位孩子想把自己的動物放到同一個有利位置。
2. 一位孩子的位置失敗，先拒絕使用自己的後援，另一位提出跨玩家後援。
3. 雙動物同時擲骰後，骰子仍能清楚歸屬各自卡片。
4. 一位孩子提供建議，但不能替另一位決定屬性或消耗後援。
5. 未出場的後援在結局取得不擲骰的後勤工作。

## 五、共同記錄格式

每局依序記錄：

1. 玩家模型與任務開場。
2. 8 隻動物完整數值及合作模式技能屬性。
3. 六個前線位置、兩隻後援與配置理由。
4. 每關實際選用屬性、數值、區間是否符合、技能是否加骰、最終骰池。
5. 初始骰、成功數、單向支援的來源與重擲結果。
6. 後援資格判斷、是否使用、替換位置、後援骰池與最後結果。
7. Boss 剩餘盾數、門檻、兩邊最終分布與整局勝負。
8. 每關一句孩子方法、DM 接話與結果旁白。
9. 決策次數、需要 DM 澄清的次數、未使用動物如何在結局出現。
10. 遊戲結束後才列出阻擋問題、非阻擋問題與純觀察。

## 六、不得自行補規則

遇到原文未定義的情況時：

1. 停在該步驟，記為「規則缺口」。
2. 採最保守、最少新增機制的臨時解讀完成該局。
3. 報告中同時列出採用的臨時解讀與另一個合理解讀。
4. 不得把自己的修正當成 v0.4 正式規則。

## 七、進入 v0.5 的門檻

只有以下項目都成立，才能把修正版標記為 v0.5 實玩候選：

- 6 局都能完成，或所有中止都有明確且可修正的規則缺口。
- 相同輸入可以得到相同骰池、支援、後援與 Boss 結果。
- 沒有需要在遊戲中翻查三個以上段落才能處理的常見操作。
- 支援與後援不會產生循環、重複計算或成功數相加歧義。
- 雙孩子的卡片所有權與後援消耗權可以直接執行。
- 所有修正都能以最小文字變更完成，不加入新資源或新子系統。
