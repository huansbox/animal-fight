# 團隊任務 v0.6：Agent 模擬協定

> 日期：2026-07-17
> 狀態：3 Agents 共 6 局模擬完成；結果見 `2026-07-17-team-mission-v06-agent-simulation-review.md`
> 規則來源：`2026-07-17-team-mission-v06-progressive-reveal-rules.md`
> 目的：3 位 agents 各跑 1 局自然局與 1 局邊界局，共 6 局
> 基準結算：後援 A「取較高、不相加」
> 影子比較：後援 B「成功數補足缺口」

## 一、共同規則

1. 模擬途中不得修改或補寫規則。
2. 基準局一律依後援 A 結算；只有遇到後援資格或使用時，才同步計算 B 版是否改變資格、結果或勝負。
3. B 版是影子分支，不改變 A 版後續的主要選角與亂數序列。
4. 前三關必須逐關揭露。玩家在選角時只能使用已公開的當前任務、Boss 提示、卡面與目前三區狀態；不得利用尚未揭露任務最佳化。
5. 所有動物與數值必須取自 `game/digital/data/animals.json`。
6. 每顆骰 4–6 算成功。合作模式技能只固定加 1 顆骰，不使用卡面 bonus 數字。
7. 自然局抽牌與骰子使用同一個 Python `random.Random(seed)`：先 `rng.sample(animals, 8)`，再依實際擲骰順序持續 `rng.randint(1, 6)`。
8. 雙動物同時擲骰時，為了重播，亂數先依 A 位置、再依 B 位置分配；實體故事仍視為同時擲出。
9. 自然局由玩家模型做決策，不能偷看後續骰子。
10. 邊界局可以預先固定骰序，但必須在該關擲骰前公開，不能看到結果後補改。

## 二、固定任務：暴雨森林救援隊

### 開場公開資訊

故事：

> 暴雨喚醒洪水巨獸，牠正朝森林避難區前進。救援隊要趕到動物村，沿途處理危機，最後突破水牆並保護避難區。

Boss：洪水巨獸，開始有 3 面護盾。

| Boss 位置 | 故事工作 | 區間 | 可選屬性 |
|---|---|---|---|
| 4A | 突破水牆 | 大格 7–9 | 力量／攻擊 |
| 4B | 保護避難動物 | 剛好格 4–6 | 防禦／智慧 |

開場不公開前三關，也不宣布各盾數的成功門檻。

### 逐關揭露內容

只有完成上一關敘事後，才揭露下一列：

| 順序 | 任務 | 故事危機 | 區間 | 可選屬性 |
|---:|---|---|---|---|
| 1 | 尋找安全道路 | 積水與倒木切斷道路，要找出能通行的路線 | 剛好格 4–6 | 速度／智慧 |
| 2A | 固定藤橋 | 洪水搖晃藤橋，需要穩住橋身 | 大格 7–9 | 力量／防禦 |
| 2B | 接近幼獸 | 受驚幼獸躲在橋邊，需要輕輕靠近 | 小格 1–3 | 速度／攻擊 |
| 3 | 搬運藥品 | 易碎藥瓶不能摔，也不能抱得太用力 | 小格 1–3 | 力量／速度 |

六個位置的小格、剛好格、大格各兩次。

## 三、自然局分工

### Agent A：9 歲策略型

- seed：`7101`
- 固定抽牌核對：眼鏡蛇、海豚、竹節蟲、巨嘴鳥、紅袋鼠、行軍蟻、黑豹、無尾熊。
- 玩家會比較當前任務骰池，也會參考開場 Boss 提示保留候選，但不預先知道後面三關。
- 若多個配置同樣合理，需說明選擇理由，不得用後續骰子決定。

### Agent B：5 歲故事直覺型

- seed：`7202`
- 固定抽牌核對：白頭海鵰、北極狐、長頸鹿、袋熊、象海豹、海象、斑馬、蜜獾。
- 玩家優先用動物外觀與故事理由選角。卡住時，DM 可以指出兩個合適候選，但不說最佳骰池。
- 必須記錄故事直覺與區間相反時，DM 如何白話說明而不否定孩子。

### Agent C：兄弟合作型

- seed：`7303`
- 固定抽牌與所有權：
  - 哥哥：牧羊犬、海龜、鴕鳥、獨角仙。
  - 弟弟：紅袋鼠、蘭花螳螂、海豚、大猩猩。
- 8 張完全公開；自由討論但由卡片主人決定。
- 不強制平均出場，也不強制雙動物各派一隻。
- 必須記錄實際主要位置分布、是否出現同一孩子派兩隻，以及另一人是否仍有參與。

## 四、指定邊界局

邊界局可由 agent 從 132 隻中選擇適合的真實動物，並在報告列出完整數值、技能、骰池與預先骰序。

### Agent A：規則順序與 A／B 難度差

必須覆蓋：

1. 第一關主要動物 0 成功，接受失敗；該動物進再挑戰區，不能救援自己剛失敗的同一關。
2. 第二關先出現 `2＋0`，完成一次單向支援但重擲仍失敗，再讓第一關的再挑戰動物擔任後援；第二次出場後直接進休息區。
3. 第三關成功，使 Boss 以 1 面盾進場。
4. Boss 初始 `1＋1`，A 版需要後援單獨取得 2 成功；B 版只差 1 成功。指定一隻只有 1 顆骰但骰出成功的候選，形成「該候選在 A 版個別不具資格／在 B 版可補足並通關」的明確分支；不要求 A 版全隊沒有其他合格後援，需另外列出其他候選。
5. 驗證 B 分支不改變 A 基準局後續亂數或正式勝負。

### Agent B：第一次後援失敗與第二次再挑戰

必須覆蓋：

1. 第一關主要動物 0 成功，呼叫一隻待命區動物後援；該後援也得到 0 成功，因此進再挑戰區。
2. 第二關由另一組主要動物出場，任務失敗後，讓第一關那隻後援以再挑戰身分第二次出場並成功；使用完第二枚後援標記後直接進休息區。
3. 後面任務即使失敗，也不能再呼叫後援。
4. 另以固定查表案例驗證一般雙動物 `0＋0` 不能呼叫一隻後援通關。
5. 失敗旁白不得責怪孩子的動物、屬性或故事方法。

### Agent C：彈性所有權與牌數安全

必須覆蓋：

1. 前三關的四個主要位置由哥哥派 3 隻、弟弟派 1 隻；第二關兩個主要位置都由哥哥提供。
2. 前兩次失敗各由弟弟派一隻全新的待命動物跨玩家後援，證明同一玩家可以提供兩隻全新後援，且各次皆由弟弟本人決定。
3. 抵達第三關前，不使用任何個人主要／後援配額的臨時限制。
4. 第三關由弟弟派主要動物，抵達 Boss 時應剛好剩下哥哥、弟弟各一隻待命動物，能完成兩個 Boss 主要位置。
5. 整局六個主要位置最終分布為哥哥 4、弟弟 2；不因不平均而判違規。
6. 自由討論可以多次提出建議，但不能替卡片主人消耗後援或指定屬性。

## 五、共同報告格式

每局必須依序記錄：

1. 玩家模型、開場實際公開資訊。
2. 8 隻動物完整數值、合作技能屬性及雙孩子所有權。
3. 每關揭露前的待命區、再挑戰區與休息區。
4. 當關翻開的故事、區間與兩個屬性；確認玩家先前不知道。
5. 主要動物選擇、卡片主人、選用屬性、故事理由與是否參考 Boss。
6. 每隻骰池計算、初始骰、初始成功與單向支援。
7. 每隻主要動物最終成功數及「完成／再挑戰」判斷。
8. 整體任務結果、護盾變化與具體故事後果。
9. 後援來源、所有權、資格、骰池、骰序及後援標記消耗。
10. A 版正式結果與 B 版影子結果；若相同也要寫明。
11. 本關結束後三區狀態。
12. 決策、DM 提示、兄弟協調、失敗旁白與未出場後勤。
13. 阻擋問題、非阻擋問題、純觀察。

## 六、通過門檻

v0.6 只有同時滿足下列條件，才可進入劇情模板設計：

1. 六局都能在不補規則的情況下完成。
2. 玩家在每關選角時沒有使用尚未揭露任務資訊。
3. 待命、再挑戰、完成三區的所有卡片去向唯一，沒有同一卡同時存在兩區。
4. 剛失敗動物不會救援同一關，再挑戰動物第二次出場後不會再次回到再挑戰區。
5. 每關最多一隻、整局最多兩隻後援，標記消耗一致。
6. 六個主要位置在最多兩次全新後援後仍有足夠待命動物完成。
7. 單向支援先於後援，且沒有循環或因後援產生新支援。
8. A、B 兩種後援資格與結算差異可以清楚重算。
9. 雙孩子不需要固定出場配額，也不會因 `4＋2` 分布讓流程停住。
10. 故事失敗不責怪孩子，後來再挑戰成功也不回溯舊護盾。

Agent 報告只能證明規則可模擬，不能取代 5 歲、9 歲與兄弟實際遊玩的理解度、等待時間與情緒觀察。
