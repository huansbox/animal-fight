# 團隊任務 v0.7：固定劇情模板

> 日期：2026-07-17
> 狀態：已更新為 Boss 寶物版本；三套既有劇情已完成遷移
> 規則依據：[`2026-07-19-team-mission-v07-treasure-boss-design.md`](2026-07-19-team-mission-v07-treasure-boss-design.md)；逐關揭露與出場規則沿用 [`2026-07-17-team-mission-v06-progressive-reveal-rules.md`](2026-07-17-team-mission-v06-progressive-reveal-rules.md)
> 目的：讓後續劇情可以批次產生，但每套仍維持逐關揭露、能力平衡與卡通式起承轉合

已完成的固定劇情與建議實玩順序見 [`scenarios/README.md`](scenarios/README.md)。

## 一、模板要解決的事

每套任務要同時做到：

1. 開場只讓孩子知道大目標與 Boss 類型，不先解完整排陣。
2. 前三關逐關翻開，每次只處理眼前危機。
3. Boss 提示提供輕量規劃，但不鎖定動物。
4. 小格、剛好格、大格都有自然的故事理由，不把小數值說成比較弱。
5. 孩子能描述「哪隻動物，如何使用所選能力完成工作」。
6. 成功、失敗、單向支援、後援與再挑戰都能接成故事。
7. 任務失敗不會卡死後續劇情；0–3 件寶物都能進 Boss，不依賴某一張指定寶物才能繼續。

## 二、固定骨架

基礎版一律使用：

| 階段 | 主要位置 | 故事作用 | 揭露時機 |
|---|---:|---|---|
| 任務簡報 | 0 | 說明大目標、Boss 與最後兩類工作 | 開場 |
| 第一關 | 1 | 抵達障礙；讓隊伍開始行動 | 完成簡報後 |
| 第二關 | 2 | 核心救援；兩隻動物分工合作 | 第一關敘事後 |
| 第三關 | 1 | Boss 前最後阻礙；通常需要精準或克制 | 第二關敘事後 |
| Boss | 2 | 一邊處理主要威脅，一邊保護目標 | 開場提示工作；抵達時才結算已取得寶物 |

六個位置必須符合：

- 小格 1–3：2 個位置。
- 剛好格 4–6：2 個位置。
- 大格 7–9：2 個位置。
- 每個位置提供 2 個可選屬性。
- 不指定必須使用哪一隻動物。

## 三、劇情資料欄位

### 1. 基本資料

| 欄位 | 說明 |
|---|---|
| `id` | 英文或數字穩定識別碼 |
| `title` | 孩子聽得懂的任務名稱 |
| `archetype` | 災難救援／調查解謎／遠征運送 |
| `tone` | 緊張、神祕、溫暖、冒險等 |
| `setting` | 森林、城市、雪山、海岸等 |
| `goal` | 一句話大目標 |
| `boss` | 最後主要威脅；可以是生物、天候、機關或自然現象 |
| `rescue_target` | 要保護、找回或送達的對象 |

### 2. 開場簡報

必須包含：

- 2–3 句 DM 開場。
- Boss 身分與目前造成的危機。
- Boss A、B 兩個工作的位置名稱、區間、兩個可選屬性。
- Boss 原始門檻固定為兩邊各至少 2 成功；不必在劇情資料內重複設計不同門檻。
- 一句提醒：「這只是提醒，不用現在決定或保留哪隻動物。」

不得包含：

- 前三關的具體故事。
- 三件寶物會抽到哪一件，以及寶物骰的結果。
- 對某張動物卡的直接暗示。

### 3. 一般任務

每關至少有：

| 欄位 | 說明 |
|---|---|
| `reveal` | 翻開時 DM 念的一句危機 |
| `positions` | 一個或兩個主要位置 |
| `job` | 這個位置要完成的具體工作 |
| `range` | 小格／剛好格／大格 |
| `attributes` | 兩個可選屬性 |
| `range_reason` | 為什麼這關需要小、剛好或大的數值 |
| `child_prompt` | 只針對已選屬性問「牠要怎麼用這個能力完成工作？」的情境化問法 |
| `fallbacks_by_attribute` | 每個可選屬性各準備兩個方法；DM 只念已選屬性的兩個，不是標準答案 |
| `success` | 成功後的一句旁白 |
| `treasure_draw` | 整體任務成功後找到寶物、請孩子抽 1 張的通用銜接句 |
| `zero_partial` | 0 成功仍留下的一小部分成果 |
| `failure_consequence` | 整體失敗後可帶到結局的具體後果 |
| `transition` | 不論成敗都能進下一關的一句銜接 |

雙動物任務額外要求：

- A、B 是不同工作，不是兩隻一起做同一件模糊的事。
- 兩個工作都能各自判斷是否完成。
- A 完成、B 未完成時，旁白仍說得通。
- 單向支援能描述成 A 做完後幫 B，或 B 做完後幫 A。

### 4. Boss

Boss 需要：

- 抵達時依取得 0–3 件寶物可套用的開場句。
- A、B 各自的工作、區間、屬性與故事理由。
- 三件寶物依固定順序結算時，可以插入劇情的簡短旁白。
- 兩邊都有成功但總量不足時的增援說法。
- 後援前先固定另一邊，算出被補位置最低需求；後援骰池必須有機會單獨達到需求。
- 後援基準 A 明寫「原位置與後援取較高、不相加」。
- 後援只與寶物結算後的主要位置結果比較，不重新取得任何寶物效果。
- 原位置 0 成功時進再挑戰區；原位置至少 1 成功但總量不足時仍進休息區。
- 成功結局。
- 失敗結局。
- 能插入前三關未完成後果的結局句型。

## 四、DM 可念與操作分層

每個結果分成兩部分，避免把規則長句一起念給 5 歲孩子：

- `DM 可念`：1–2 句具體故事，只說發生什麼事。
- `DM 操作`：移卡、抽寶物、寶物結算、是否可後援、最低需求與重擲限制；由家長看，不必逐字念。

若舊稿仍將兩者寫在同一項，至少要用「故事」與「操作」兩行分開。

## 五、統一旁白句型

以下句型讓批次劇情不必為每種骰況重寫完整段落。

### 主要動物 0 成功

> `{主要動物}` 已經 `{局部成果}`，但 `{環境阻力}` 讓牠沒能完成自己的工作。牠先到再挑戰區，之後可以擔任一次後援。

### 別的動物接手成功

> `{主要動物}` 留下的進度沒有消失；`{後援動物}` 接著用 `{後援方法}` 完成剩下工作。整體任務成功，但 `{主要動物}` 仍保留再挑戰資格。

### 後援也失敗

> `{後援動物}` 也完成了 `{局部成果}`，可是 `{新的環境阻力}` 讓這次補救仍未成功。保留後果，隊伍繼續前進。

### 再挑戰動物第二次出場

成功：

> `{再挑戰動物}` 想起上次的經驗，這次完成了 `{目前工作}`。牠的兩次出場都用完，接著到休息區。

失敗：

> `{再挑戰動物}` 再努力一次，留下 `{局部成果}`，但這次補救沒有成功。牠的兩次機會已用完，接著到休息區。

### 後援標記已用完

> 再挑戰區的動物仍願意幫忙，但全隊兩次呼叫機會都已用完；這關保留目前結果，隊伍繼續前進。

### 不回溯

> `{動物}` 後來的表現只完成現在的工作；前一關留下的 `{舊後果}` 不會改變，也不會回頭補抽當時沒有取得的寶物。

### 一般任務成功並抽寶物

> 隊伍完成了這段工作，也在現場找到一件可以在最後幫上忙的寶物。請從剩下的寶物中抽 1 張，先翻開放在桌上，抵達 Boss 再使用。

## 六、區間故事設計指南

### 小格 1–3

小數值不是比較弱，而是工作需要克制、輕巧、緩慢或不造成破壞。

可用理由：

- 力量：輕拿、不要壓壞、只推一點。
- 速度：慢慢靠近、不能滑倒、避免驚動。
- 攻擊：輕敲、精細切開、不要傷到旁邊對象。
- 防禦：身體不要太厚重、需要穿過狹窄處；這類理由較難，能不用就避免。
- 智慧：不要過度推理、照簡單信號行動；這類理由較抽象，基礎劇情能不用就避免。

### 剛好格 4–6

工作需要控制與平衡，太小做不到、太大容易過頭。

可用理由：穩定搬運、適量施力、控制節奏、觀察但不分心、保護且保持移動。

### 大格 7–9

工作需要明顯的力量、速度、攻擊、防禦或複雜判斷。

可用理由：推開巨大障礙、追上快速目標、突破厚牆、承受衝擊、解開多層線索。

## 七、批次產生檢查清單

### 機制

- [ ] 結構固定為單動物、雙動物、單動物、雙動物 Boss。
- [ ] 六個位置的小格、剛好格、大格各 2。
- [ ] 每個位置恰有 2 個屬性。
- [ ] Boss 兩個工作在開場公開，原始門檻固定為 `2＋2`，不設計劇情專屬門檻表。
- [ ] 前三關沒有在開場洩漏。
- [ ] 每關不依賴某隻指定動物。
- [ ] 雙動物兩邊能分別判斷。
- [ ] 整體失敗後仍能進下一關。
- [ ] 每個一般任務成功後都抽 1 張未取得寶物；失敗不抽且不回溯。
- [ ] Boss 依手電筒、魔鏡、背包的固定順序結算，再擲主要骰、支援與後援。

### 故事

- [ ] 小格理由不是「這隻比較弱」。
- [ ] 孩子方法沒有標準答案，也不改骰數。
- [ ] 每個位置的兩個屬性各有兩個可選提示，DM 只念已選屬性的提示。
- [ ] 0 成功保留局部成果，不把失敗歸咎孩子。
- [ ] 後援成功不抹掉原動物的再挑戰資格。
- [ ] 再挑戰成功不回溯前一關。
- [ ] 未取得的寶物不因後來成功而補發。
- [ ] Boss 成功與失敗都能交代前三關後果。
- [ ] 未出場動物能得到不擲骰的後勤工作。
- [ ] 若 8 隻都已出場，可由休息區或再挑戰區動物承接結局後勤，不改變勝負。

### 語言

- [ ] DM 每次揭露控制在 1–2 句。
- [ ] 5 歲孩子不需要理解抽象術語才可選擇。
- [ ] 首次出現區間時，說明為小格／剛好格／大格。
- [ ] 不用死亡、永久受傷或責怪角色的結果。

## 九、後續隨機任務擴充

基礎版先使用固定三關。若真人實玩確認流程好玩，再把任務拆成三類牌庫：

1. 抵達障礙：單動物。
2. 核心救援：雙動物。
3. Boss 前障礙：單動物。

每類各抽一張，不直接從全部任務任抽三張。屆時每張任務卡還需增加 `stage`、`task_type`、`range_slots`、`story_tags`、`compatible_boss_tags` 與通用轉場欄位；目前不實作。
